## 系统架构

```
┌──────────────────────────────────────────────────────────────┐
│                     xgb_extrema_selector.py                  │
│                     选股器（每日运行生成信号）                   │
│  - 加载历史数据 → 训练模型 → 生成买卖信号                        │
│  - 输出 signal_df 到 AlphaLab                                │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│                   xgb_extrema_live_trading.py                │
│                     实盘/模拟盘交易程序                        │
│  - 支持三种模式：回测 / 模拟盘 / 实盘                           │
│  - 连接行情网关，执行交易信号                                   │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│              LiveAlphaEngine (实盘/模拟盘引擎)                 │
│  ├─ 实时行情接收 (Tick/Bar)                                  │
│  ├─ 订单管理 (send_order / cancel_order)                     │
│  ├─ 持仓同步 (Position events)                               │
│  ├─ 资金查询 (Account events)                               │
│  └─ 信号执行 (execute_daily_signals)                        │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│                    VNPY MainEngine                           │
│  ├─ EventEngine (事件分发)                                    │
│  ├─ OmsEngine (订单管理)                                     │
│  └─ XtGateway (迅投交易网关 - 可选)                           │
│       - connect() 连接交易服务器                              │
│       - send_order() 发送真实委托（仅实盘模式）                 │
│       - query_position() 查询持仓                            │
└──────────────────────────────────────────────────────────────┘
```

## 三种交易模式

| 模式 | 命令 | 行情 | 订单 | 资金 | 适用场景 |
|------|------|------|------|------|----------|
| **回测** | `--mode backtest` | 历史数据 | 模拟撮合 | 虚拟 | 策略验证、参数调优 |
| **模拟盘** | `--mode paper` | 实时行情 | 本地撮合 | 虚拟 | 真实环境验证、学习 |
| **实盘** | `--mode live` | 实时行情 | 真实交易所 | 真实 | 真实交易 |

### 模式对比

#### 回测模式
- 使用历史 K 线数据
- 本地模拟成交
- 结果可重复，适合策略验证

#### 模拟盘模式
- 使用实时行情（Tick）
- 本地模拟撮合（以最新价成交）
- 检查资金/持仓是否足够
- 计算手续费和盈亏
- **不发送真实订单到交易所**

#### 实盘模式
- 使用实时行情
- 订单发送到真实交易所
- 同步券商账户持仓和资金
- 受券商风控限制

## 快速开始

### 1. 环境准备

```bash
# 安装依赖
pip install -e .[alpha]

# 安装迅投研数据服务（可选，用于实时行情）
pip install vnpy_xt

# 配置数据库（用于存储信号和历史数据）
# 默认使用 PostgreSQL，可在代码中修改 SETTINGS
```

### 2. 生成交易信号

选股器每日收盘后运行，生成次日的交易信号：

```bash
# 运行选股器
python xgb_extrema_selector.py
```

信号会保存到 `./lab/csi300/signals/` 目录下。

### 3. 回测验证

```bash
# 回测模式（离线验证策略逻辑）
python xgb_extrema_live_trading.py --mode backtest
```

### 4. 模拟盘验证

```bash
# 模拟盘模式（使用实时行情，本地撮合）
python xgb_extrema_live_trading.py --mode paper --capital 1000000
```

### 5. 实盘交易

```bash
# 实盘模式（真实订单发送到交易所）
python xgb_extrema_live_trading.py --mode live --gateway XT --capital 1000000
```

## 配置说明

### 数据服务配置

在 `xgb_extrema_live_trading.py` 中配置：

```python
# 数据库配置（存储历史数据）
SETTINGS["database.name"] = "postgresql"
SETTINGS["database.host"] = "localhost"
SETTINGS["database.port"] = "5432"
SETTINGS["database.database"] = "vnpy"
SETTINGS["database.user"] = "vnpy"
SETTINGS["database.password"] = "vnpy"

# 数据服务配置（迅投研）
SETTINGS["datafeed.name"] = "xt"
SETTINGS["datafeed.username"] = "client"
SETTINGS["datafeed.password"] = ""
```

### 实盘网关配置

在 `LiveTrader.connect_gateway()` 中配置迅投账号：

```python
def connect_gateway(self) -> bool:
    if self.gateway_name == "XT":
        from vnpy_xt import XtGateway
        self.main_engine.add_gateway(XtGateway, self.gateway_name)
        
        # 迅投登录配置（修改这里）
        setting = {
            "账号类型": "股票账号",  # 或 "信用账号"
            "账号": "YOUR_ACCOUNT_ID",  # 替换为你的账号
        }
        
        self.main_engine.connect(setting, self.gateway_name)
```

### 策略参数配置

在 `LiveTrader.setup_strategy()` 中配置：

```python
strategy_setting = {
    "min_days": 1,              # 最小持有天数
    "profit_threshold": 0.03,   # 盈利止盈阈值（3%）
    "cash_ratio": 0.95,         # 现金利用率（95%）
    "min_volume": 100,          # 最小交易单位（100股）
    "open_rate": 0.0005,        # 买入费率（万分之五）
    "close_rate": 0.0015,       # 卖出费率（千分之一点五）
    "min_commission": 5,        # 最低手续费（5元）
    "price_add": 0.0,           # 下单价格偏移（滑点）
}
```

```

XGBoost 极值策略实盘交易
```

## 交易流程

### 每日交易流程

```
09:15  开盘前准备
       ├── 运行选股器生成当日信号（如未运行）
       └── 启动交易程序，加载信号

09:30  开盘
       ├── 接收实时行情（Tick）
       └── 检测买入信号，执行买入

14:55  收盘前
       ├── 检测卖出信号（止盈/止损）
       └── 执行卖出

15:00  收盘
       └── 程序自动停止（或保持运行等次日）
```

### 信号执行逻辑

```python
# 买入条件
if signal == 1 and 当前无持仓:
    计算买入数量 = 可用资金 / 目标股票数 / 当前价格
    发送买入订单

# 卖出条件
if signal == -1 and 当前有持仓 and 持有天数 >= min_days:
    发送卖出订单

# 止盈条件
if 当前盈利 >= profit_threshold:
    发送卖出订单
```

## 常见问题

### Q1: 模拟盘和实盘有什么区别？

**A:** 主要区别：
- **模拟盘**：使用真实行情，但订单在本地模拟撮合，不发送到交易所，不产生真实盈亏
- **实盘**：订单发送到真实交易所，产生真实成交和盈亏

建议先用模拟盘验证策略在真实行情下的表现，再切换到实盘。

### Q2: 如何切换不同的交易网关？

**A:** 目前支持迅投研网关（XT）。如需其他网关（如 CTP），需要：
1. 安装对应的 vnpy 网关包（如 `vnpy_ctp`）
2. 在代码中添加网关配置
3. 使用 `--gateway CTP` 参数指定

### Q3: 信号文件在哪里？如何更新？

**A:** 信号文件保存在 `./lab/csi300/signals/` 目录下，文件名为选股时指定的 `name` 参数（如 `300_xgb_extrema.parquet`）。

每日收盘后运行选股器更新信号：
```bash
python xgb_extrema_selector.py
```

### Q4: 实盘交易有风险吗？

**A:** **有！实盘交易使用真实资金，可能产生亏损。**

建议：
1. 先用回测和模拟盘充分验证策略
2. 实盘初期使用小资金测试
3. 确保理解策略逻辑和风险
4. 设置止损和仓位控制

### Q5: 如何停止交易程序？

**A:** 按 `Ctrl+C` 优雅停止：
- 自动撤销所有未完成订单
- 保存交易统计
- 关闭网关连接

### Q6: 交易时间段是什么？

**A:** 默认交易时间：
- 开始：`09:30`（开盘）
- 结束：`14:55`（收盘前5分钟）

可在 `LiveAlphaEngine` 中修改：
```python
self.trading_start_time = time(9, 30)
self.trading_end_time = time(14, 55)
```

### Q7: 为什么我的订单没有成交？

**A:** 可能原因：

**模拟盘模式：**
- 资金不足（检查可用资金）
- 持仓不足（卖出时检查持仓）
- 无行情数据（检查数据服务连接）

**实盘模式：**
- 价格偏离市价太远（限价单可能不成交）
- 券商风控限制
- 非交易时间
- 股票停牌

### Q8: 如何查看详细的交易日志？

**A:** 日志输出在控制台，也可配置保存到文件。在代码中添加：
```python
import logging
logging.basicConfig(
    filename='trading.log',
    level=logging.INFO,
    format='%(asctime)s - %(levelname)s - %(message)s'
)
```

## 风险提示

**⚠️ 重要提示：**

1. **量化交易存在风险**，过往业绩不代表未来表现
2. **实盘交易前务必充分测试**，建议先用回测和模拟盘验证
3. **确保理解策略原理**，不要盲目跟单
4. **合理控制仓位**，不要投入无法承受损失的资金
5. **定期检查策略表现**，及时调整或停止

**使用本软件即表示您理解并接受以上风险。**
